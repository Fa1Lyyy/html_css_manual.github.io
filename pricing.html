Основные элементы HTML

1. Элемент <HTML>

Является главным элементом документа. Все остальные элементы содержатся внутри <html>...</html>. Все, что находится не в этом элементе, не воспринимается браузером как код и не обрабатывается.

2. Элемент <head>

Раздел <head>...</head> содержит техническую информацию о странице: подключенные файлы (css, js), описание, название страницы , кодировку и т.д.

3. Элемент <title>

Обязательным элементом тега <head> является <title>. Текст, размещенный внутри элемента <title>, отображается в заголовке браузера.

4. Элемент <META>

в теге <meta> содержится информация о кодировке сайта:

<meta charset="UTF-8"> <meta name="viewport"content=width=device-width, initial-scale=1.0>

5. Элемент <style>

В этот элемент задаются стили, которые используются на странице. Для задания стилей в HTML-документе используется CSS. Атрибуты для <style> : media,nonce,type .

6. Элемент <link>

С помощью тега линк к html документу подключаются например файлы стилей CSS, иконки и т.д. Пример:

<link rel="stylesheet"href="style.css"type=text/css>

7. Элемент <script>

Элемент <script> позволяет присоединять к документу различные сценарии. Текст сценария может располагаться либо внутри этого элемента, либо во внешнем файле. Если текст сценария расположен во внешнем файле, то он подключается с помощью атрибутов элемента (async,charset,crossorigin,defer,nonce,src,type). В основной используются JS файлы.

8. Элемент <body>

В разделе <body> располагается все содержимое документа. Конкретрно код содержимого страницы, к примеру верхняя навигационная панель

9. Атрибуты ссылок

download,href,hreflang,media,ping,rel,target,type

10. Элемент <img>

Элемент <img> представляет изображение и его резервный контент, который добавляется с помощью атрибута alt, лучше всего располагать в блоке div или p

Форматы графических файлов:JPEG,GIF,PNG,APNG,SVG,BMP,ICO

11. Элемент <map>

Элемент <map> служит для представления графического изображения в виде карты с активными областями. Активные области определяются по изменению вида курсора мыши при наведении на них.

12. Элемент <area>

Элемент <area> описывает только одну активную область в составе карты изображений на стороне клиента. Если одна активная область перекрывает другую, то будет реализована первая ссылка из списка областей.

Атрибуты для area - alt,coords,download,href,hreflang,media,rel,shape,target,type

13. <article>

Элемент <article> представляет собой законченное произведение в док, стр, прил или сайте. Может содержать внутри другие элементы <article>, которые по содержанию имеют близкое отношение к содержанию внешней статьи. Если на странице присутствует только одна статья с заголовком и текстовым содержимым, она не нуждается в обёртке элементом <article>

14. Элемент <section>

Элемент <section> не является универсальным контейнером. Когда элемент нужен только для стилизации или для удобства написания сценариев,используйте вместо этого элемент <div>. Общее правило заключается в том, что элемент <section> уместен только в том случае, если содержимое элемента будет явно указано в схеме документа.

15. Элемент <nav>

Элемент <nav> в основном используется для создания навигационных панелей.

16. Элемент <footer>

Элемент чаще всего находится в нижней части страницы и содержит в себе контакную информацию, сведениями об авторских правах, ссылками на условия использования

17. Элемент <picture>

вместе с элементом <source> используется для предоставления множественных источников изображения. Это дает браузеру возможность выбора оптимальной версии изображения, в зависимости от плотности пикселей экрана, размера области просмотра, формата изображения и других факторов.

18. Элемент <source>

позволяет указывать несколько альтернативных источников изображений для элемента <img> или несколько альтернативных медиа-ресурсов для элементов <audio> или <vidio>

19. Элемент <frame>

используется для встраивания другого HTML-документа в текущий, при этом он полностью изолирован от JavaScript и CSS родительского элемента.

20. Элемент <embed>

используется для отображения внешних ресурсов или интерактивного контента на странице. Элемент <embed> не подключает альтернативные ресурсы,поэтому если браузер не находит подходящий плагин при попытке найти и создать экземпляр ресурса, то он выдаст сообщение о неподдерживаемом формате.

21. Элемент <object>

редставляет внешний ресурс, который, в зависимости от типа ресурса, будет рассматриваться либо как изображение, либо как вложенный контекст просмотра, либо как внешний ресурс, который будет обрабатываться плагином.

22. Элемент <video>

используется для воспроизведения видео или фильмов, а также аудиофайлов с субтитрами.

23. Элемент <audio>

элемент представляет собой звук или аудиопоток

24. Элемент <track>

позволяет явно указывать внешние текстовые ресурсы для медиа-элементов. Сам по себе он ничего не представляет.

25. Элемент <table>

представляет табличные данные и не используется для разметки страницы. Если указан, значением атрибута должна быть либо пустая строка, либо значение «1»

26. Элемент <thead>

участвует в макете таблицы, представляя собой блок строк, состоящий из заголовков столбцов таблицы.

27. Элемент <form>

редставляет собой гиперссылку, которой можно управлять с помощью набора связанных элементов управления, значения которых можно отправить на сервер для обработки.

28. Элемент <select>

представляет собой элемент управления для выбора среди набора параметров.

29. Элемент <progress>

выводит на экран виджет, который отображает индикатор выполнения задачи. С его помощью пользователи могут отслеживать ход выполнения длительной операции. Индикатор выполнения может либо показать приблизительный процент завершения, либо указать, что операция выполняется.

30. Элемент <details>

представляет виджет раскрытия информации, который используется, чтобы показать или скрыть дополнительную информацию в конкретной области интерфейса. Не подходит для примечаний.

31. Элемент <summary>

представляет заголовок или небольшое пояснение для остального содержимого родительского элемента <details>. Щелчок по элементу <summary> переключает состояние родительского элемента в открытое и закрытое состояния.

32. Элемент <dialog>

Контекст, в котором этот элемент может быть использован - где ожидается потоковое содержимое.

33. Элемент <template>

используется для объявления фрагментов HTML-разметки, которые можно клонировать и вставлять в документ с помощью сценария. При загрузке страницы элемент <template> не отображается.

34. Элемент <canvas>

пределяет пустую растровую область для рисования. Он не имеет собственных объектов, только инструкции Canvas API о том, что рисовать на любом отдельном кадре. Canvas API в основном ориентирован на 2D-графику. Его можно использовать для рисования графики, анимации, игровой графики, визуализации данных, обработки фотографий и видео в реальном времени.

<a> = Создаёт гипертекстовые ссылки.

<br> = Перенос текста на новую строку.

<h> = заголовок текста , имеет виды h1,h2,h3,h4,h5,h6.

<b> = задает полужирный шрифт

<em> = текст курсивом

<small> = Уменьшает шрифт на единицу по отношению к обычному тексту.

<strong> = задает полужирнй шрифт, указывающий браузеру на важность текста.

<sub> = Используется для создания нижних индексов. Сдвигает текст ниже ур строки, уменьшая его размер.

<sup> = Используется для создания степеней. Сдвигает текст выше ур строки, уменьшая его размер.

<mark> = Используется для выделения фрагментов текста в справочных целях, окрашивая блок желтым цветом.

<code> = Служит для выделения фрагментов программного кода. Отображает текст моноширинным шрифтом.

<kbd> = Отмечает фрагмент текста как вводимый пользователем с клавиатуры.

<samp> = Применяется для выделения результата, полученного в ходе выполнения программы.

<var> = Выделяет имена переменных, текст отображается курсивом.

<pre> = Позволяет вывести текст на экран, сохранив изначальное форматирование.

<abbr> = Применяется для форматирования аббревиатур.Подчеркивается пунктирной линией.

<bdo> = Используется для изменения направления текста.

<blockquote> = Выделяет цитаты внутри документа, выделяя его отступами и переносами строк.

<q> = используется для выделения коротких цитат, текст берется в кавычки

<p> = разбивает текст на абзацы

<hr> = Используется для разделения контента на странице. Отображается в виде горизонтальной линии.

<address> = представляет контактную информацию о человеке или организации

<hr> = представляет собой тематический разрыв на уровне абзаца

<pre> = представляет собой блок предварительно отформатированного текста, в котором структура представлена типографскими соглашениями, а не элементами.

<blockquote> = редставляет содержимое, цитируемое из другого источника, необязательно со ссылкой на источник цитирования

<ol> = представляет собой список элементов, где элементы были изначально упорядочены таким образом, что изменение порядка изменило бы смысл документа. Элементами списка являются элементы <li>

<ul> = редставляет собой список элементов, где порядок элементов не важен, то есть когда изменение порядка не приведет к существенному изменению смысла документа.

<li> = представляет собой элемент списка. Если родительским элементом является элемент <ol>.

<dl> = представляет список описаний, состоящий из нуля или более групп термин-описание. Термин представлен элементом <dt>, описание — элементом <dd>

<figure> = редставляет автономное содержимое (необязательно с подписью), являющееся самостоятельным элементом основного потока.

<figcaption> =представляет заголовок или легенду для остального содержимого родительского элемента <figure>.

<main> = включает основное содержимое элемента <body> документа или приложения и исключает содержимое, которое повторяется на страницах сайта/приложения, таких как ссылки для навигации по сайту, информация об авторских правах, логотипы сайта и баннеры, а также поисковые формы (за исключением случаев, когда основной функцией документа или приложения является поисковая форма).

<small> = представляет собой примечания, набранные мелким шрифтом.

<s> = представляет содержимое, которое больше не является точным или актуальным.

<cite> = представляет собой ссылку на творческое произведение.

<dfn> = используется для выделения термина.

<ruby> = позволяет пометить один или несколько фрагментов текста ruby-аннотациями.

<rb> = представляет основные сегменты текста ruby-аннотации. Является дочерним элементом элемента ltruby>

<rt> = спользуется для разметки аннотации к тексту, выводя ее сверху или снизу над текстом.

<rtc> = помечает вложенный в него текст как дополнительную аннотацию.

<rp> = выводит альтернативный текст в случае если браузер не поддерживает элемент <ruby>.

<data> = представляет свое содержимое вместе с машиночитаемой формой этого содержимого в атрибуте value.

<time> = представляет свое содержимое вместе с машиночитаемой формой этого содержимого в атрибуте datetime.

<i> = представляет справочные выделения отдельных слов и фрагментов текста. Используется для выделения терминов, формулировок, определений, фраз из иностранных языков.

<bdi> = представляет фрагмент текста, который должен быть изолирован от остального текста для двунаправленного форматирования текста.

<span> = Элемент <span> сам по себе ничего не значит, но может быть полезен при использовании вместе с глобальными атрибутами, например, class, lang или dir. Он представляет свои дочерние элементы.

<wbr> = используется для разрыва длинной строки, чтобы текст можно было переносить в удобочитаемом виде.

<ins> = представляет собой добавления к документу, например, одного абзаца

<del> = представляет собой удаление из документа.

<tr> = частвует в макете таблицы, представляя собой строку ячеек в таблице.

<td> = представляет собой ячейку данных в таблице.
